# 모바일 기기 접속 가이드 (Mobile Connection Guide)

이 문서는 Mac에서 실행 중인 로컬 개발 서버에 휴대폰 등 모바일 기기를 연결하여 확인하는 방법을 설명합니다.

## 사전 준비 (Next.js 설정)

외부 기기에서 접속하려면 Next.js 서버가 모든 네트워크 인터페이스(0.0.0.0)에서 응답하도록 실행되어야 합니다.

```bash
# 프로젝트 폴더에서 실행
npm run dev -- -H 0.0.0.0
```

> [!NOTE]
> 포트가 이미 사용 중인 경우 `3001` 등으로 자동 변경될 수 있으니 터미널에 표시된 포트 번호를 확인하세요.

---

## 방법 1: 동일 Wi-Fi망 사용 (추천)

Mac과 휴대폰이 동일한 공유기(Wi-Fi)에 연결되어 있을 때 사용하는 가장 일반적인 방법입니다.

1.  **Mac의 IP 주소 확인**: 터미널에서 `ifconfig` 명령어를 통해 Wi-Fi(보통 `en0`)의 IP를 확인합니다. (예: `192.168.2.32`)
2.  **휴대폰 접속**: 브라우저 주소창에 `http://[Mac-IP]:3000` 을 입력합니다.
    - 예: `http://192.168.2.32:3000`

---

## 방법 2: Mac 인터넷 공유 (AP 모드)

외부 Wi-Fi 보안 환경으로 인해 기기간 통신이 막혀 있거나, 공유기가 없을 때 사용합니다. Mac이 직접 Wi-Fi 신호를 쏘는 방식입니다.

### 1. Mac 설정
1.  **시스템 설정 > 일반 > 공유** 메뉴로 이동합니다.
2.  **인터넷 공유** 항목의 `i` 버튼을 눌러 설정을 확인합니다.
    - **공유할 연결**: 현재 인터넷을 사용 중인 서비스 (이더넷 등)
    - **다음 사용**: **Wi-Fi** 체크
3.  **Wi-Fi 옵션**에서 네트워크 이름(SSID)과 암호를 설정합니다.
4.  **인터넷 공유** 스위치를 **'켬'**으로 바꿉니다.

### 2. 휴대폰 접속
1.  휴대폰 Wi-Fi 설정에서 위에서 만든 Mac의 네트워크에 연결합니다.
2.  브라우저에서 Mac의 내부 게이트웨이 주소로 접속합니다.
    - 보통 주소: **`http://192.168.3.1:3000`** (포트 번호 확인 필수)

---

## 트러블슈팅

### 1. 접속이 되지 않을 때 (무한 로딩)
- **방화벽 확인**: Mac의 **시스템 설정 > 네트워크 > 방화벽**이 '켬'으로 되어 있다면 잠시 **'끔'**으로 변경한 후 테스트해 보세요.
- **포트 번호 확인**: 터미널 로그에 `Port 3000 is in use, trying 3001 instead`와 같은 메시지가 있다면 `:3001`로 접속해야 합니다.

### 2. Localtunnel 보안 페이지가 뜰 때
- 외부 URL(`loca.lt`)을 사용할 경우, 자신의 **Public IP**를 패스워드로 요구할 수 있습니다.
- 터미널에서 `curl ifconfig.me`를 입력하여 나오는 숫자를 입력하면 접속됩니다.
