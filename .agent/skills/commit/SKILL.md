# Commit Skill 지침

사용자가 'commit' 또는 이와 유사한 요청을 할 경우 다음 절차를 따르세요.

## 실행 절차

1.  **진행 상황 분석**: 마지막 커밋 이후의 변경 사항을 `git diff` 등을 통해 분석합니다.
2.  **progress.md 업데이트**:
    - 루트 디렉토리의 `progress.md` 파일을 읽습니다. (파일이 없으면 생성합니다.)
    - 날짜와 시간을 포함하여 현재까지의 주요 변경 사항, 해결된 이슈, 추가된 기능을 요약하여 추가합니다.
3.  **Git 커밋**:
    - 모든 변경 사항을 스테이징합니다: `git add .`
    - 변경 내용을 요약한 의미 있는 커밋 메시지를 작성합니다.
    - 커밋을 수행합니다.
4.  **Git 푸시**: 원격 저장소로 `push`를 수행합니다.

## 주의 사항
- `progress.md`는 가장 최근 작업이 상단에 오도록 기록하거나 연대순으로 깔끔하게 유지하세요.
- 커밋 메시지는 한글 또는 영문으로 명확하게 작성하세요.
